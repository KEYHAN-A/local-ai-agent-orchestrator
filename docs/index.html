Local AI Agent Orchestrator

Planner, coder, and reviewer agents on your machine — LM Studio, SQLite, memory-safe model swaps.

OpenAI-SDK-native pipeline (no heavy agent framework). Optimized for local inference, tight context windows, and one big model in RAM at a time.

Three-phase loop

Architect decomposes plans into micro-tasks. Coder writes code with tools. Reviewer approves or sends feedback.

Persistent state

SQLite WAL queue survives crashes; retry limits; full run log for audits.

Memory gate

After unload, waits for macOS unified memory to reclaim pages before the next load — less swap / SSD wear.

Configurable

factory.yaml + CLI lao: URL, RAM, paths, per-role model keys.

Reviewer (v1.1+)

Reasoning models (e.g. R1 distill) can think before they answer — LAO strips think blocks and finds APPROVED / REJECTED on any line.

Clean layout (v1.1.1)

SQLite and generated code live under .lao/: .lao/state.db and .lao/workspaces/<plan-stem>/ per plans/*.md. No root workspace/ folder.

Install & run

git clone https://github.com/KEYHAN-A/local-ai-agent-orchestrator.git
cd local-ai-agent-orchestrator
python -m venv .venv && source .venv/bin/activate
pip install -e .
lao init && cp factory.example.yaml factory.yaml
# Edit factory.yaml — set model keys from `lms ls`
lao health
lao run
