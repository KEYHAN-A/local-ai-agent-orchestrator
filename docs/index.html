Local-first · LM Studio · Your silicon

Run a local coding agent factory with a seamless operator flow

Local AI Agent Orchestrator (LAO) turns markdown plans into shipped code with a planner → coder → reviewer loop and a unified UX across lao, lao init, lao configure-models, and lao run. Keep state in SQLite, recover smoothly after interruptions, and keep progress traceable with optional per-plan Git.

Capabilities

Built to squeeze serious work out of local models

Three-agent pipeline

Architect breaks plans into micro-tasks. Coder edits files with tools. Reviewer approves or sends structured rework—without bloated agent-framework scaffolding.

SQLite backbone

WAL-mode queue survives crashes; retries, run logs, and token tallies stay queryable while jobs churn overnight.

Memory-aware swaps

Loads one big LLM at a time, unloads the rest, and waits for unified memory to settle—less swap thrash on Apple Silicon when models are tens of GB.

Semantic retrieval

Optional embedder (e.g. Nomic) narrows file context before coding so small local windows still hit the right files.

New in v1.3

Git you can trust

Per-plan repos get LAO_PLAN.md, LAO_TASKS.json, and commits named lao(coder): … / lao(reviewer): … so progress is traceable and easy to revert.

New in v2.0

Unified interactive CLI

Home/status, guided init, model remap, and run dashboard now share one coherent TTY experience with guided next actions.

Flow

From setup to resumed execution

Open home

Run lao to see environment readiness and jump into init, model setup, health, or execution.

Guided setup

lao init creates factory.yaml interactively and can bootstrap an initial plan in one flow.

Run pipeline

lao run executes planner/coder/reviewer with a Rich dashboard and per-plan workspace mapping.

Pause and continue

Queue state persists in SQLite. Restarting lao run recovers interrupted tasks and continues work.

Quick start

Install and run in minutes

pip install local-ai-agent-orchestrator
lao
# choose "init workspace" and follow guided setup
lao health
lao run

Existing project workflow: clone/open your repo into ./MyRepo/ and use plans/MyRepo.md so LAO works directly in that codebase folder.
