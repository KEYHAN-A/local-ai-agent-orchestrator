Local-first · LM Studio · Your silicon

Run a coding agent factory on your own hardware, around the clock

Local AI Agent Orchestrator (LAO) turns markdown plans into shipped code using a planner → coder → reviewer loop—no cloud API required at runtime. Swap heavy models safely, keep state in SQLite, and let your Mac or workstation put local LLMs to work while you sleep.

Capabilities

Built to squeeze serious work out of local models

Three-agent pipeline

Architect breaks plans into micro-tasks. Coder edits files with tools. Reviewer approves or sends structured rework—without bloated agent-framework scaffolding.

SQLite backbone

WAL-mode queue survives crashes; retries, run logs, and token tallies stay queryable while jobs churn overnight.

Memory-aware swaps

Loads one big LLM at a time, unloads the rest, and waits for unified memory to settle—less swap thrash on Apple Silicon when models are tens of GB.

Semantic retrieval

Optional embedder (e.g. Nomic) narrows file context before coding so small local windows still hit the right files.

New in v1.3

Git you can trust

Per-plan repos get LAO_PLAN.md, LAO_TASKS.json, and commits named lao(coder): … / lao(reviewer): … so progress is traceable and easy to revert.

Operator-friendly CLI

factory.yaml plus flags for URLs, RAM, models, and paths. Rich dashboard on a TTY, or --plain for logs in CI.

Flow

From markdown to merged changes

Drop a plan

Add plans/Feature.md. LAO hashes content and opens a matching project folder next to plans/.

Decompose

The planner model emits a JSON task list; tasks land in SQLite with dependencies and priorities.

Code & review

The coder model uses file tools inside the plan workspace; the reviewer model verdicts APPROVED or REJECTED with feedback loops.

Ship & rewind

Optional Git commits after each phase; roll back a single agent step or replay history with standard Git tools.

Quick start

Install and run in minutes

pip install local-ai-agent-orchestrator
lao init && cp factory.example.yaml factory.yaml
# Edit factory.yaml — point lm_studio_base_url and model keys (`lms ls`)
lao health
lao run

From source: clone the repo, pip install -e ., same commands. Use lao --plan plans/YourPlan.md --single-run run for one-shot runs.
